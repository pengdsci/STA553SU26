1. Theme Fundamentals & Element Hierarchy

🔷 Complete vs. Incomplete Themes

Understanding theme types
  • Complete themes override all plot elements — use as base
  • Built-in: theme_grey() (default), theme_bw(), theme_minimal(), theme_classic(), theme_void(), theme_light(), theme_dark()
  • Incomplete themes modify only specific elements — use + theme(...)
  • Example: + theme(axis.text.x = element_text(angle = 45, hjust = 1))
  • Use %+replace% to completely replace inherited elements
base_size %+replace% inheritance

🏗️ Plot Element Hierarchy

Full inheritance tree
  • plot → panel (background, border, grid.major, grid.minor, spacing, ontop)
  • plot → axis (title, text, ticks, ticks.length, line — each for x, y, top, right)
  • plot → legend (background, title, text, key, key.size, margin, spacing, position, justification, direction)
  • plot → strip (background, text, placement, switch — for faceted plots)
  • plot → plot.title, .subtitle, .caption, .tag, .margin, aspect.ratio
panel.grid strip.placement aspect.ratio

2. Step-by-Step Custom Theme Creation

📝 Steps 1–2: Function Skeleton & Background

Foundation of every custom theme
  • Step 1: theme_custom <- function(base_size = 12, base_family = "") { }
  • Always start with theme_minimal() or theme_bw() + %+replace%
  • Step 2: plot.background = element_rect(fill = "white", colour = NA)
  • panel.background = element_rect(fill = "grey95", colour = NA)
  • panel.border = element_rect(fill = NA, colour = "grey80", linewidth = 0.5)
base_size base_family skeleton

📏 Steps 3–4: Grid Lines & Axis Elements

Grid and axis configuration
  • Step 3: panel.grid.major = element_line(colour = "grey85", linewidth = 0.3)
  • panel.grid.minor = element_line(colour = "grey90", linewidth = 0.15, linetype = "dotted")
  • Step 4: axis.line = element_line(colour = "black", linewidth = 0.5)
  • axis.ticks = element_line(colour = "grey50") + axis.ticks.length = unit(2, "mm")
  • axis.title = element_text(face = "bold", margin = margin(t = 10))
linetype unit() dotted

🏷️ Steps 5–6: Legend & Plot Titles

Legend positioning and text hierarchy
  • Step 5: legend.position = "bottom" | "right" | "none" | c(0.9, 0.9)
  • legend.background = element_rect(fill, colour, linewidth), legend.key = element_rect()
  • Step 6: plot.title = element_text(size = rel(1.4), face = "bold", hjust = 0.5)
  • plot.subtitle = element_text(size = rel(1.1), colour = "grey40", hjust = 0.5)
  • plot.caption = element_text(size = rel(0.8), hjust = 1) + plot.margin = margin(20,20,20,20)
legend.justification plot.tag

📋 Step 7: Facet Strips & Final Assembly

Facet strip styling and complete theme
  • strip.background = element_rect(fill = "grey80", colour = "grey50")
  • strip.text = element_text(size = rel(1.1), face = "bold", colour = "grey20")
  • strip.placement = "outside" (or "inside")
  • Combine all 7 steps into one theme() call inside %+replace%
  • Return the complete theme object — test with test_plot + theme_custom()
facet_wrap facet_grid strip.switch

3. Complete Theme Examples — Publication, Dashboard & Dark

📄 Publication-Ready Theme

Clean, academic, print-optimized
  • Base: theme_bw(base_size = 14, base_family = "serif")
  • White panel, black border (linewidth = 0.8), major grid only
  • Black axis ticks (linewidth = 0.5), axis.ticks.length = unit(2, "mm")
  • Legend at bottom with bordered background, bold strip text on grey90
  • Left-aligned title (hjust = 0), right-aligned caption, generous margins
Academic serif print

📊 Dashboard Minimal Theme

Compact, modern, business-oriented
  • Base: theme_void(base_size = 12) + faint major grid lines only
  • Simple axis lines (colour = "grey60", linewidth = 0.3), minimal ticks
  • Floating legend: legend.position = c(0.9, 0.9), legend.justification = c(1, 1)
  • Transparent legend background: fill = alpha("white", 0.8), no key border
  • Compact plot margins: margin(10, 10, 10, 10), centered title
Business alpha() floating

🌙 Dark Presentation Theme

High-contrast for slides & dark UIs
  • Base: theme_minimal(base_size = 16), dark backgrounds throughout
  • plot.background = element_rect(fill = "grey20"), panel = "grey25"
  • Grid lines visible: major "grey40", minor "grey35"
  • Light text: axis "grey90", titles "white", bold face for emphasis
  • Legend on dark: fill = "grey30", colour = "grey40", key = "grey25"
Slides contrast presentation

🎨 Theme Variant with Accent Color

Extending a base theme with branding
  • Create theme_base() first, then extend with %+replace%
  • theme_colorful <- function(base_size = 12, accent_color = "#2196F3") { }
  • Apply accent to: axis.title, plot.title, strip.background
  • Strip text becomes white for contrast on colored background
  • Test variants: theme_colorful(accent_color = "#E74C3C")

4. Theme Inheritance, Best Practices & Advanced Techniques

🔁 %+replace% vs + — Deep Dive

Understanding theme composition
  • %+replace% completely replaces the entire element subtree — no inheritance from parent
  • + adds or overrides specific properties while keeping others from the parent theme
  • Example: theme_bw() %+replace% theme(panel.grid.minor = element_blank())
  • Use %+replace% when building a complete custom theme from scratch
  • Use + for incremental tweaks: theme_custom() + theme(axis.text = element_text(size = 14))
composition override

✅ Best Practices Checklist

Production-ready theme development
  • Always provide base_size and base_family parameters
  • Use rel() for scalable font sizes — never hardcode absolute sizes
  • Handle missing fonts gracefully with tryCatch() or font availability checks
  • Test with: scatter plots, bar charts, faceted plots, multi-legends, and empty data
  • Include Roxygen documentation: @param base_size, @param base_family, @return, @export
  • Ensure WCAG 2.1 AA contrast ratios for accessibility (4.5:1 for text)
accessibility testing docs

🧪 Responsive & Conditional Themes

Dynamic theming techniques
  • Responsive scaling: theme_responsive <- function(ratio = 1) { element_text(size = base_size * ratio) }
  • Conditional grid: if (!grid) panel.grid.major = element_blank()
  • Conditional border: if (border) panel.border = element_rect(...)
  • Combine with ggplot2::theme_get() / theme_set() for global defaults
  • Use theme_update() to modify the active theme globally in a session
theme_set() theme_update() dynamic

🐛 Debugging & Testing Themes

Tools for verifying theme behavior
  • Inspect elements: my_theme$panel.grid.major — returns the element definition
  • Use ggplot2::calc_element("panel.grid.major", my_theme) for computed values
  • Create a standard test plot with points, lines, facets, and legends
  • Compare themes side-by-side with patchwork or gridExtra
  • Check for missing elements: setdiff(names(theme_grey()), names(my_theme))
