STA 553 Data Visualization

Core Programmatic Skills for Including Additional Graphic Information to Existing Plots

Modern R + Markup workflows • From data processing to interactive storytelling • Effective design principles

📌 1. Foundational Mindset: Processed Information > Raw Data

Visualization is a compressed encoding of extracted information. In R + Quarto/RMarkdown, processing and visualization coexist inseparably. Without mutate() / summarise(), no effective plot exists.

raw_data → filter() → mutate() → group_by() → summarise() → visualize()
💡 Core principle: If you cannot write the transformation step that extracts the information, you cannot design an effective visual representation.
🔧 Data Processing Mastery

🎨 2. Static Enhancement Techniques

Math Expressions Image Embedding Layout Composition

Semantic Annotation: ggplot2::annotate(parse=TRUE) for formulas & statistical results.
Images & Branding: patchwork::inset_element(), cowplot::draw_image() to add logos or context.
Layout hierarchy: patchwork / facet_wrap() to guide viewer attention across multi-panel narratives.

🖌️ Static Design Proficiency

🖱️ 3. Interactive Storytelling Techniques

Tooltips (hover) → reveal values/deltas via ggplotly(aes(text=...))
Popup texts (click modals) → methodological notes using shiny::modalDialog()
Linked subplots → shared brushing with plotly::subplot(shareX=TRUE) + crosstalk for pattern discovery.

✨ Interactivity & Exploration
